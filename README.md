# JSON-CAD pipeline

Минимальный генератор макросов КОМПАС из JSON-команд.

Проект держится только на базовых универсальных CAD-примитивах. В ядре не должно быть частных команд под конкретные предметы или готовые бытовые объекты.

## Pipeline

```text
JSON
-> validator
-> command emitter
-> KOMPAS .m3m macro
```

LLM позже должна будет собирать детали из универсальных команд, а не вызывать частные шаблоны объектов.

## Запуск

```powershell
python src/main.py examples/box.json
```

Если запустить без аргумента, используется `examples/box.json`.

## Структура

- `src/main.py` - точка входа.
- `src/json_loader.py` - чтение JSON.
- `src/validator.py` - проверка допустимых команд и параметров.
- `src/commands.py` - перевод JSON-команд в операции сборщика макроса.
- `src/kompas_macro_builder.py` - низкоуровневый сборщик текста `.m3m`.
- `src/macro_generator.py` - запись готового макроса.

## Основные Команды

Разрешены только универсальные геометрические команды:

- `create_box` - создать прямоугольный брусок.
- `cut_box` - вырезать прямоугольным бруском.
- `create_prism` - создать призму по произвольному 2D-контуру.
- `cut_prism` - вырезать по произвольному 2D-контуру.
- `create_triangle` - создать треугольную призму.
- `cut_triangle` - вырезать треугольной призмой.
- `create_cylinder` - создать цилиндр.
- `cut_cylinder` - вырезать цилиндром.

Команды под конкретные готовые детали не являются ядром и не должны добавляться в генератор. Такие вещи должны собираться в JSON из базовых команд.

## Размещение

У базовых команд можно использовать универсальное размещение:

- `placement.origin` - поставить первую фигуру в координату.
- `attach.target` - прикрепить фигуру к ранее созданной фигуре.
- `attach.face` - сторона целевой фигуры: сейчас поддержаны `top`, `bottom`, `front`.
- `attach.position` - пока поддержан `center`.
- `attach.offset` - смещение по плоскости стороны `[dy, dz]`.

Это не отдельные шаблоны деталей, а способ располагать базовые фигуры.

Пример:

```json
{
  "id": "roof",
  "type": "create_triangle",
  "size": [70, 28, 34],
  "attach": {
    "target": "body",
    "face": "top",
    "position": "center"
  }
}
```

## Пример

```json
{
  "version": "0.1",
  "units": "mm",
  "part_name": "simple_box",
  "commands": [
    {
      "id": "box_1",
      "type": "create_box",
      "origin": [0, 0, 0],
      "size": [100, 60, 20],
      "plane": "YOZ",
      "extrude": "middle",
      "select_point": [0, -10, 0]
    }
  ]
}
```

## Что Можно Делать Примерами

В `examples/` можно хранить демонстрационные сборки: буквы, звезды, пластины с отверстиями, цилиндры с вырезами. Но это должны быть именно композиции из базовых команд, а не новые специальные типы команд.

## Наблюдения По Макросам

Рабочие макросы КОМПАС используют повторяющийся паттерн:

1. подключение COM API;
2. получение активного 3D-документа;
3. создание эскиза;
4. выбор плоскости;
5. рисование контура через `ksLineSeg`, `ksCircle`, `ksArcByPoint`;
6. завершение эскиза;
7. создание `o3d_bossExtrusion` или `o3d_cutExtrusion`;
8. выбор контура через `SelectByPoint`;
9. настройка глубины;
10. `obj.Create()`.

Поэтому новые команды добавляем только тогда, когда они являются базовой геометрией или базовой операцией размещения.
